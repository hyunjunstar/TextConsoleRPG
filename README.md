# 1. 게임 컨셉 및 구상

## 1.1 게임 개요
C++로 개발하는 텍스트 기반 RPG 게임이다.  
플레이어는 캐릭터를 생성하고 10개의 스테이지를 통과하며 몬스터와 전투한다.  

스테이지를 클리어할 때마다 유물을 획득하여 캐릭터를 성장시키고,  
경험치를 쌓아 레벨업하며, 최종 보스를 물리치면 게임 클리어이다.



## 1.2 학습 목표

| 목표 | 설명 | 적용 사례 |
|------|------|----------|
| 프로그래밍 기본기 | 변수, 함수, 조건문, 반복문 활용 | 전투 흐름, 스테이지 루프 |
| 객체지향 설계 | 구조 설계 | BasicMonster, Factory 패턴 |
| 데이터 관리 | STL 활용 | vector, unique_ptr |
| 클린 코드 | 유지보수 가능한 구조 | 단일 책임, 데이터 기반 |



## 1.3 초기 구상 vs 최종 구현

| 항목 | 초기 구상 | 최종 구현 |
|------|----------|----------|
| 스테이지 | 3단계 | 10단계 + 보스 |
| 몬스터 | 클래스 분리 | BasicMonster 통합 |
| 유물 | 고정 | 랜덤 선택 |
| 전투 | 단방향 | 양방향 + 확률 |



## 1.4 팀원 및 역할

| 이름 | 역할 | 담당 파일 |
|------|------|-----------|
| 백록담 | 메인 / 스테이지 | Main.cpp |
| 김남윤 | 플레이어 시스템 | Player.h / Player.cpp |
| 김현준 | 몬스터 시스템 | BasicMonster.h / BasicMonster.cpp |
| 조유현 | 전투 / 팩토리 | BattleManager / MonsterFactory / Artifact |
| 정태민 | 유물 기획 | Artifact 데이터 설계 |
| 전지원 | UI 및 텍스트 연출 / 색상 | UI.h / UI.cpp |



## 1.5 기술 스택

- C++
- STL (vector, unique_ptr)
- Windows Console API
- Git / GitHub

# 2. 시스템 설계 및 구현

## 2.1 몬스터 시스템

### 구현 방식
- BasicMonster 단일 클래스로 통합
- 몬스터 특성을 데이터로 관리
- MonsterFactory로 생성 분리

### 특징
- 클래스 추가 없이 몬스터 확장 가능
- 스테이지별 스케일링 적용
- 보스 몬스터 별도 처리



## 2.2 전투 시스템

### 전투 흐름
1. 플레이어 공격  
2. 몬스터 사망 체크  
3. 몬스터 반격 (스킬 → 실패 시 평타)  



## 2.3 유물 시스템

- 30개 유물  
- 랜덤 선택 시스템  
- 티어 기반 확률  



## 2.4 레벨업 시스템

| 항목 | 내용 |
|------|------|
| 경험치 | 30 + 스테이지 × 10 |
| 보상 | HP +20, ATK +5 |
| 최대 레벨 | 10 |



## 2.5 추가 시스템

| 시스템 | 내용 |
|--------|------|
| 포션 | 스테이지 클리어 시 획득, 사용 시 HP 50 회복 |
| 상태 확인 | 전투 중 현재 스탯 확인 (턴 소모 없음) |
| 색상 UI | 콘솔 색상으로 상황 구분 (보스/보상/시스템) |
| ASCII 아트 | 몬스터 등장 및 전투 연출 |

# 3. 밸런스 시뮬레이션

## 3.1 시뮬레이션 개요
- Python 기반 자동 시뮬레이션 진행  
- 총 1000회 플레이 데이터 분석  
- 목표 클리어율: 40% ~ 60%  



## 3.2 밸런싱 과정

| 버전 | 조정 내용 | 클리어율 | 결과 |
|------|----------|----------|------|
| v1 | 초기 상태 | 40.0% | 기준 충족 |
| v2 | 일반 몬스터 버프, 보스 너프 | 76.6% | 난이도 과도하게 낮음 |
| v3 | 일반 몬스터 추가 버프, 보스 일부 복원 | 53.2% | 최종 |



## 3.3 최종 결과

- 클리어율: **53.2%**  
- 평균 도달 스테이지: 9.2  
- 중간 보스 사망률: 26.6%  
- 최종 보스 사망률: 20.0%

# 4. 설계 구조 및 원칙

## 4.1 파일 구조

| 파일 | 역할 |
|------|------|
| BasicMonster | 몬스터 통합 관리 |
| MonsterFactory | 몬스터 생성 및 스케일링 |
| BattleManager | 전투 로직 |
| Artifact | 유물 시스템 |
| Player | 플레이어 |
| Main | 게임 루프 |
| UI | 출력 및 연출 |



## 4.2 적용된 설계 원칙

- Factory Pattern 적용  
- Data Driven 설계  
- Single Responsibility Principle  
- unique_ptr 사용으로 메모리 관리
